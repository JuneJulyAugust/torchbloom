# Functions As Machines

## Core Idea

A function is a machine: feed it an input, follow its rule, and get an output.
UDL writes this as expressions like `y = f[x]` or `y = f[x, phi]`.

## Why It Matters In UDL

Supervised learning models are functions. Training changes the parameters of the
function, not the input-output idea itself.

## High-School Bridge

The line `y = 2x + 3` is already a model. The input is `x`, the output is `y`,
and the numbers `2` and `3` control the slope and intercept.

## Ready For UDL When

- You can plug a number into a formula and compute the output.
- You can say which symbols are inputs and which are fixed parameters.
- You can explain why changing a parameter changes the whole function.
